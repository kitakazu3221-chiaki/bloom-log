# 頭皮記録Webアプリ 設計書

## 1. プロジェクト概要

毎日頭皮の写真を撮影・記録し、薄毛の進行や改善を視覚的に確認できるWebアプリケーション。

---

## 2. ターゲットユーザー

- 1人用（シングルユーザー）
- PCで記録・管理を行うユーザー
- スマホを撮影カメラとして利用

---

## 3. 主要機能

### 3.1 撮影機能
- スマホのカメラ映像をWebRTC経由でPCにリアルタイム配信
- QRコードによるスマホ・PC間のセッション連携
- PCの大画面で映像プレビューを確認しながら撮影
- 前回写真のオーバーレイ表示（位置合わせガイド）
- ガイド円・十字線の表示による撮影位置の再現性確保
- PC側の撮影ボタン（カウントダウンタイマー対応）

### 3.2 記録・保存機能
- 撮影した写真をローカルフォルダに保存（File System Access API）
- 撮影日時・部位（頭頂部・前頭部・側頭部など）のメタデータ記録
- 補助データの記録（使用シャンプー、育毛剤、睡眠、ストレスなど）

### 3.3 閲覧・比較機能
- カレンダー形式での撮影履歴一覧
- 日付スライダーによるBefore/After比較
- タイムライン表示
- 部位ごとの経過確認

---

## 4. 撮影フロー

```
1. PCでアプリを開く
      ↓
2. QRコードが表示される
      ↓
3. スマホでQRコードを読み取る
      ↓
4. スマホのカメラが起動 → 映像をPCに送信開始
      ↓
5. PCの画面に映像 + 前回写真オーバーレイ + ガイド円が表示される
      ↓
6. ユーザーはPC画面を見ながらスマホを頭上で位置合わせ
      ↓
7. PC側で撮影ボタンを押す（またはカウントダウン後自動撮影）
      ↓
8. 写真がローカルフォルダに保存され、記録に追加される
```

---

## 5. システム構成

```
スマホ（カメラ）
    │
    │ WebRTC（映像ストリーム）
    │ ※映像・写真データはサーバーを経由しない
    ▼
PC（ブラウザ）
    │
    │ プレビュー表示
    │ ガイドオーバーレイ
    │ 撮影ボタン
    │
    ▼
ローカルフォルダ（File System Access API）
    └── 写真ファイル
    └── メタデータ（JSON）

※ シグナリングサーバー（Node.js）はWebRTC接続確立のためだけに使用
```

## 8. 撮影再現性への対策

| 課題 | 対策 |
|------|------|
| 角度のズレ | 前回写真のオーバーレイ表示 |
| 位置のズレ | ガイド円・十字線の表示 |
| 照明のバラつき | 明るさが極端な場合に警告表示 |
| 距離のズレ | 撮影スタイル（腕を伸ばすなど）を初回に登録し毎回表示 |

---

## 9. プライバシー・セキュリティ

- 写真・映像データはシグナリングサーバーを経由しない（P2P通信）
- すべてのデータはローカルフォルダに保存（クラウド不使用）
- サーバーはWebRTC接続確立のシグナリングのみに使用
